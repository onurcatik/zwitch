Introduction

MERN Stack ile Twitch Klon Uygulaması Oluşturma

Genel Bakış

Bu eğitimde, MERN stack (MongoDB, Express.js, React.js ve Node.js) kullanarak bir Twitch klon uygulaması oluşturacağız. Bu uygulama, kanal tarama, akış izleme, sohbet özellikleri, kullanıcı kimlik doğrulama ve bildirimler gibi Twitch'in temel işlevlerinin çoğunu kopyalayacaktır. Uygulamanın uygulanmasının kesin, titiz ve yazılım geliştirme alanındaki en iyi uygulamalara uygun olmasını sağlayacağız.

İşlevler

  1. Kanal Tarama: Kullanıcılar bir dizi kanalı tarayabilir.
  2. Kanal Detayları: Her kanalın avatarını, başlığını, açıklamasını ve çevrimiçi durumunu görüntüleyebilir.
  3. Kanal İzleme: Kullanıcılar, kanal çevrimiçi ise akışları izleyebilir, kanalın açıklamasını ve takip etme düğmesini görebilir.
  4. Sohbet Sistemi: Her kanalın bir sohbet özelliği olacak ve ziyaretler arasında mesajlar saklanacaktır.
  5. Kullanıcı Kimlik Doğrulama: Kullanıcılar kaydolabilir, giriş yapabilir ve kimlik doğrulaması yapılırsa kullanıcı adlarını görebilir.
  6. Kanal Yönetimi: Kullanıcılar, kullanıcı adlarını, başlıklarını, avatarlarını ve açıklamalarını ayarlayarak kanallarını yönetebilir.
  7. Yayın Entegrasyonu: Her kullanıcının, OBS gibi yayın yazılımına bağlanmak için benzersiz bir yayın anahtarı olacaktır.
  8. Bildirimler: Kullanıcılar, örneğin yanlış giriş kimlik bilgileri gibi hatalar için bildirimler alacaklar.

Ön Koşullar

Başlamadan önce, aşağıdakilerin yüklü olduğundan emin olun:

Kullanıcı Kimlik Doğrulama

Backend

Bir models dizini ve kullanıcı şeması için bir User.js dosyası oluşturun.

const mongoose = require('mongoose');
const bcrypt = require('bcryptjs');

const UserSchema = new mongoose.Schema({
    username: { type: String, required: true, unique: true },
    email: { type: String, required: true, unique: true },
    password: { type: String, required: true },
});

// Kaydetmeden önce şifreyi hashleyin
UserSchema.pre('save', async function(next) {
    if (!this.isModified('password')) {
        return next();
    }
    const salt = await bcrypt.genSalt(10);
    this.password = await bcrypt.hash(this.password, salt);
    next();
});

module.exports = mongoose.model('User', UserSchema);

Kullanıcı kaydı ve giriş için bir routes dizini oluşturun.

const express = require('express');
const bcrypt = require('bcryptjs');
const jwt = require('jsonwebtoken');
const User = require('../models/User');

const router = express.Router();

// Register
router.post('/register', async (req, res) => {
    const { username, email, password } = req.body;

    try {
        const user = new User({ username, email, password });
        await user.save();
        res.status(201).send('User registered');
    } catch (error) {
        res.status(400).send(error.message);
    }
});

// Login
router.post('/login', async (req, res) => {
    const { email, password } = req.body;

    try {
        const user = await User.findOne({ email });
        if (!user) {
            return res.status(400).send('Invalid credentials');
        }

        const isMatch = await bcrypt.compare(password, user.password);
        if (!isMatch) {
            return res.status(400).send('Invalid credentials');
        }

        const token = jwt.sign({ userId: user._id }, 'your_jwt_secret', { expiresIn: '1h' });
        res.json({ token });
    } catch (error) {
        res.status(500).send('Server error');
    }
});

module.exports = router;

Bu route'ları server.js dosyasına entegre edin.

const userRoutes = require('./routes/users');
app.use('/api/users', userRoutes);

// LoginForm.js
import React, { useState } from 'react';
import axios from 'axios';

const LoginForm = () => {
    const [email, setEmail] = useState('');
    const [password, setPassword] = useState('');
    const [error, setError] = useState('');

    const handleSubmit = async (e) => {
        e.preventDefault();
        try {
            const res = await axios.post('http://localhost:5000/api/users/login', { email, password });
            localStorage.setItem('token', res.data.token);
            setError('');
        } catch (err) {
            setError('Invalid credentials');
        }
    };

    return (
        <form onSubmit={handleSubmit}>
            <input type="email" value={email} onChange={(e) => setEmail(e.target.value)} placeholder="Email" required />
            <input type="password" value={password} onChange={(e) => setPassword(e.target.value)} placeholder="Password" required />
            <button type="submit">Login</button>
            {error && <p>{error}</p>}
        </form>
    );
};

export default LoginForm;

const express = require('express');
const ChatMessage = require('../models/ChatMessage');

const router = express.Router();

// Get chat messages for a channel
router.get('/:channelId', async (req, res) => {
    try {
        const messages = await ChatMessage.find({ channel: req.params.channelId }).populate('user', 'username');
        res.json(messages);
    } catch (error) {
        res.status(500).send('Server error');
    }
});

// Send a chat message
router.post('/:channelId', async (req, res) => {
    const { userId, message } = req.body;

    try {
        const chatMessage = new ChatMessage({
            channel: req.params.channelId,
            user: userId,
            message,
        });

        await chatMessage.save();
        res.status(201).send('Message sent');
    } catch (error) {
        res.status(400).send(error.message);
    }
});

// Chat.js
import React, { useState, useEffect } from 'react';
import axios from 'axios';

const Chat = ({ channelId, userId }) => {
    const [messages, setMessages] = useState([]);
    const [message, setMessage] = useState('');

    useEffect(() => {
        const fetchMessages = async () => {
            const res = await axios.get(`http://localhost:5000/api/chat/${channelId}`);
            setMessages(res.data);
        };

        fetchMessages();
    }, [channelId]);

    const handleSendMessage = async (e) => {
        e.preventDefault();
        if (message.trim()) {
            await axios.post(`http://localhost:5000/api/chat/${channelId}`, { userId, message });
            setMessage('');
            const res = await axios.get(`http://localhost:5000/api/chat/${channelId}`);
            setMessages(res.data);
        }
    };

    return (
        <div>
            <div>
                {messages.map((msg) => (
                    <p key={msg._id}><strong>{msg.user.username}</strong>: {msg.message}</p>
                ))}
            </div>
            <form onSubmit={handleSendMessage}>
                <input type="text" value={message} onChange={(e) => setMessage(e.target.value)} placeholder="Type a message" required />
                <button type="submit">Send</button>
            </form>
        </div>
    );
};

export default Chat;

Teknolojilere Giriş

Bu eğitimde, MERN stack'e derinlemesine bakacağız. MERN, MongoDB, Express.js, React.js ve Node.js anlamına gelir. Ek olarak, gerçek zamanlı sohbet işlevselliği için Socket.io'yu entegre edeceğiz. Bu teknolojilerin her biri, tam özellikli bir web uygulaması oluşturmak için kritik bir rol oynamaktadır. Her bileşene kısa bir giriş yaparak başlayalım.

MongoDB

MongoDB, verileri esnek, JSON benzeri bir formatta saklayan bir NoSQL veritabanıdır. Modern web uygulamalarının büyük veri hacimlerini işleyebilmesi için ölçeklenebilirlik ve esneklik sunar.

Express.js

Express.js, Node.js için bir web uygulama framework'üdür. API'ler oluşturmayı basitleştiren bir dizi middleware bileşeni sağlayarak web ve mobil uygulamalar geliştirmek için güçlü bir özellik seti sunar.

React.js

React.js, kullanıcı arayüzleri oluşturmak için bir JavaScript kütüphanesidir. Yeniden kullanılabilir UI bileşenleri oluşturmayı, karmaşık uygulamaların durumunu yönetmeyi ve veri değişikliklerinde doğru bileşenleri verimli bir şekilde güncelleyip render etmeyi sağlar.

Node.js

Node.js, Chrome'un V8 JavaScript motoru üzerine inşa edilmiş bir JavaScript runtime'dır. JavaScript ile sunucu tarafında ölçeklenebilir ağ uygulamaları oluşturmayı sağlar.

Socket.io

Socket.io, web istemcileri ve sunucuları arasında gerçek zamanlı, çift yönlü ve olay tabanlı iletişim sağlayan bir kütüphanedir. Genellikle sohbet uygulamaları, gerçek zamanlı analizler ve işbirliği araçları oluşturmak için kullanılır.

MERN Stack Uygulamasının Mimarisi

MERN stack uygulamamızın mimarisi şu bileşenlerden oluşacaktır:

  1. İstemci Tarafı (Frontend)

    • Frontend, React.js kullanılarak oluşturulacaktır. Verileri almak ve görüntülemek için backend ile API çağrıları yapacaktır.

  2. Sunucu Tarafı (Backend)

    • Backend, Node.js ve Express.js kullanılarak oluşturulacaktır. İstemci isteklerini işleyip veritabanı ile etkileşime geçecek ve API uç noktalarını sunacaktır.
    • Uygulamanın verilerini depolayıp yönetmek için MongoDB kullanılacaktır.
    • Sohbet işlevselliği gibi gerçek zamanlı iletişim sağlamak için sunucuya Socket.io entegre edilecektir.

}

MERN Stack Uygulamasında MongoDB

Giriş

Bu rehber, MERN (MongoDB, Express, React, Node.js) stack uygulamasında MongoDB kullanımıyla ilgili kapsamlı ve detaylı bir kılavuz sunmaktadır. Bu belge, yanlışlıkları veya yanlış anlamaları ele alarak bilgileri kesin ve bilimsel bir şekilde sunmayı amaçlamaktadır. Yazılım geliştiriciler için uygun bir şekilde hazırlanmıştır.

MongoDB'nin Genel Bakışı

MongoDB, verileri esnek, JSON benzeri bir format olan BSON (Binary JSON) içinde saklayan yaygın olarak kullanılan bir NoSQL veritabanıdır. Bu format, veri yapısının zamanla evrilebileceği veya veri şemasının kesin olarak tanımlanmadığı uygulamalar için özellikle uygundur. Bir MERN stack uygulamasında, MongoDB, veri depolama ve geri alma işlemlerini yöneten veritabanı katmanını oluşturur.

MongoDB'nin Temel Özellikleri

  1. Döküman Odaklı Depolama:

    • MongoDB, verileri JavaScript nesnelerine benzer dökümanlar içinde saklar. Bu format, iç içe geçmiş yapılar ve diziler için yüksek esneklik sağlar.

  2. Ölçeklenebilirlik:

    • MongoDB, sharding yoluyla yatay ölçeklemeyi destekler, bu da büyük veri hacimlerini ve yüksek okuma ve yazma işlem hacmini işleyebilecek uygulamalar için uygundur. AWS gibi bulut platformlarında kolayca dağıtılabilir ve yönetilen MongoDB hizmetleri (örneğin, Amazon DocumentDB) sunulur.

  3. Performans:

    • MongoDB'nin performansı, yerleşik önbellekleme mekanizması ile artırılır. Sık erişilen veriler bellekte saklanır, bu da tekrar tekrar disk erişimi gereksinimini azaltır ve okuma performansını artırır.

  4. Yüksek Erişilebilirlik:

    • MongoDB, otomatik replikasyon yoluyla yüksek erişilebilirlik sağlar. Replica sets, veri yedeklemesi ve failover yetenekleri sunar, böylece bazı düğümler arızalansa bile veritabanının çalışmaya devam etmesini sağlar.

  5. Esnek Şema:

React'in Temel Özellikleri

  1. Bileşen Tabanlı Mimari: React, uygulama geliştirme için modüler bir yaklaşımı teşvik eder, UI'yi izole edilmiş ve yeniden kullanılabilir bileşenlere böler. Bu, kod tabanının yönetilmesini ve bakımını kolaylaştırır.

  4. JSX Sözdizimi: React, JavaScript içinde HTML benzeri kod yazmayı sağlayan bir sözdizimi uzantısı olan JSX kullanır. JSX, kodun okunabilirliğini artırır ve bileşen yapısını görselleştirmeye yardımcı olur.

  5. Tek Yönlü Veri Bağlama: React, tek yönlü veri akışını izler, bu da uygulamayı daha öngörülebilir ve hata ayıklamayı daha kolay hale getirir. Veriler, parent bileşenlerden child bileşenlere props aracılığıyla akar.

  6. React Hooks: React 16.8'de tanıtılan hooks, fonksiyonel bileşenlerde state ve diğer React özelliklerini kullanmaya olanak tanır, daha temiz ve daha fonksiyonel bir React geliştirme yaklaşımını teşvik eder.

React'teki Temel Kavramlar

Karmaşık uygulamalara dalmadan önce, React'teki bazı temel kavramları anlamak önemlidir.

Bileşenler

Bileşenler, bir React uygulamasının yapı taşlarıdır. İki tür bileşen vardır:

  • Functional Components: JSX döndüren basit JavaScript fonksiyonlarıdır. Props kabul edebilirler ve state ve yaşam döngüsü metodları için hooks kullanırlar.
  • Class Components: React.Componenti genişleten ve JSX döndürmek için render metoduna sahip ES6 sınıflarıdır. Class bileşenleri kendi state'lerini yönetebilir ve yaşam döngüsü metodlarına sahiptir.

Functional Component Örneği:

import React from 'react';

function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

export default Greeting;

Class Component Örneği:

import React, { Component } from 'react';

class Greeting extends Component {
  render() {
    return <h1>Hello, {this.props.name}!</h1>;
  }
}

export default Greeting;

Props ve State

  • Props: Props (properties'in kısaltması), parent bileşenlerden child bileşenlere geçirilen salt okunur özelliklerdir. Verileri ve olay işleyicilerini child bileşenlere iletmek için kullanılırlar.

  • State: State, bileşene özel olan ve zamanla değişebilen yerel veri deposudur. Bir bileşenin state'i değiştiğinde, bileşen yeni state'i yansıtmak için yeniden render edilir.

Class Component'te State Kullanımı Örneği:

import React, { Component } from 'react';

class Counter extends Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  increment = () => {
    this.setState({ count: this.state.count + 1 });
  }

  render() {
    return (
      <div>
        <p>Count: {this.state.count}</p>
        <button onClick={this.increment}>Increment</button>
      </div>
    );
  }
}

export default Counter;

Hooks ile Functional Component'te State Kullanımı Örneği:

import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;

  2. Bağımlılıkları Yükleme Gerekli paketleri yükleyin, Express ve Socket.IO dahil:

Sunucu Tarafı Uygulaması

  1. Sunucuyu Oluşturma server.js adında bir dosya oluşturun ve temel bir Express sunucusu kurun:

    const express = require('express');
const http = require('http');
const socketIo = require('socket.io');

const app = express();
const server = http.createServer(app);
const io = socketIo(server);

const PORT = process.env.PORT || 3000;

server.listen(PORT, () => {
  console.log(`Server is running on port ${PORT}`);
});

  2. Socket.IO'yu Entegre Etme Socket.IO bağlantılarını yönetmek için aşağıdaki kodu ekleyin:

    io.on('connection', (socket) => {
  console.log('A user connected');

  // Gelen mesajları işle
  socket.on('message', (msg) => {
    console.log('Message received: ', msg);
    // Mesajı tüm istemcilere yayınla
    io.emit('message', msg);
  });

  // Bağlantı kesildiğinde işle
  socket.on('disconnect', () => {
    console.log('User disconnected');
  });
});

İstemci Tarafı Uygulaması

  1. İstemci Tarafı Dosyalarını Ayarlama Proje dizininde index.html adında bir dosya oluşturun ve aşağıdaki içeriği ekleyin:

    <!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Socket.IO Chat</title>
  <script src="/socket.io/socket.io.js"></script>
  <script>
    document.addEventListener('DOMContentLoaded', () => {
      const socket = io();

      socket.on('message', (msg) => {
        const messageElement = document.createElement('div');
        messageElement.textContent = msg;
        document.body.appendChild(messageElement);
      });

      document.getElementById('send').addEventListener('click', () => {
        const msg = document.getElementById('message').value;
        socket.emit('message', msg);
        document.getElementById('message').value = '';
      });
    });
  </script>
</head>
<body>
  <input type="text" id="message" placeholder="Type a message...">
  <button id="send">Send</button>
</body>
</html>

    app.get('/', (req, res) => {
  res.sendFile(__dirname + '/index.html');
});

Socket.IO'nun Temel Özellikleri

  1. Gerçek Zamanlı İletişim Socket.IO, istemci ve sunucu arasında anlık veri transferine olanak tanır. Bu, sohbet sistemleri gibi zamanında mesaj iletiminin önemli olduğu uygulamalar için kritik öneme sahiptir.

  2. Çift Yönlü İletişim Geleneksel HTTP isteklerinin aksine, Socket.IO açık bir bağlantı tutar ve istemcinin bir istek başlatmasına gerek kalmadan sunucunun güncellemeleri istemciye iletmesini sağlar.

  3. Otomatik Yeniden Bağlanma Socket.IO, bağlantı kesilmelerini algılayıp yeniden bağlanmaya çalışan yerleşik mekanizmalara sahiptir. Bu, kararsız ağ koşullarında bile bağlantının güvenilirliğini sağlar.

  4. Oda Fonksiyonu Socket.IO, soketlerin gruplandırılmasını sağlayan "odalar" kavramını destekler. Bu, birden fazla kanal veya grup yönetimi gereken uygulamalarda özellikle kullanışlıdır, örneğin farklı sohbet odaları.

    io.on('connection', (socket) => {
  console.log('A user connected');

  // Bir odaya katıl
  socket.on('joinRoom', (room) => {
    socket.join(room);
    console.log(`User joined room: ${room}`);
  });

  // Belirli bir odaya mesaj gönder
  socket.on('message', (msg, room) => {
    io.to(room).emit('message', msg);
  });

  // Bağlantı kesildiğinde işle
  socket.on('disconnect', () => {
    console.log('User disconnected');
  });
});

MERN Stack Kullanmanın Avantajlarına Dair Kapsamlı Bir Eğitim

MongoDB, Express.js, React ve Node.js'den oluşan MERN stack, sağlam ve ölçeklenebilir web uygulamaları oluşturmak için güçlü bir teknoloji kombinasyonudur. Bu eğitim, MERN stack'in kullanım avantajlarını, bileşenlerini, faydalarını ve onu destekleyen ekosistemi incelemektedir.

React ile Frontend Geliştirme

React, özellikle tek sayfa uygulamalarda, reaktivite ve performansın kritik olduğu kullanıcı arayüzleri oluşturmak için popüler bir JavaScript kütüphanesidir. MERN stack içinde React kullanmanın başlıca nedenlerinden biri, yeniden kullanılabilirlik ve sürdürülebilirlik sağlayan bileşen tabanlı mimarisidir. Geliştiriciler, kendi durumlarını yöneten kapsüllemeli bileşenler oluşturabilirler, bu da modüler ve düzenli bir kod tabanı sağlar.

React’in sanal DOM’u, verimli güncellemeler ve sorunsuz bir kullanıcı deneyimi sağlayarak render işlemlerini optimize eder. Ayrıca, Redux gibi durum yönetimi ve React Router gibi navigasyon için kullanılan geniş React kütüphaneleri ve araçları ekosistemi, yeteneklerini daha da artırır. React’in popülaritesi, bol miktarda kaynak, eğitim ve topluluk desteği sunar, bu da geliştirme zorluklarına çözüm bulmayı kolaylaştırır.

Node.js ve Express ile Backend Geliştirme

Node.js, geliştiricilerin JavaScript’i sunucu tarafında çalıştırmalarına olanak tanıyan bir çalışma ortamıdır. Bu, stack boyunca geliştirme dilinin birleştirilmesini sağlar ve öğrenme eğrisini azaltır. Node.js, V8 JavaScript motoru üzerine kuruludur ve bu da onu hızlı ve verimli kılar, özellikle I/O yoğun işlemler için.

Express.js, Node.js için minimal bir web framework'üdür ve web uygulamaları ve API'ler oluşturmak için sağlam özellikler sunar. Routing, middleware entegrasyonu ve istek yönetimini basitleştirir. Express’in esnekliği, geliştiricilerin uygulamalarını ihtiyaçlarına göre yapılandırmalarına olanak tanır ve hem küçük hem de büyük ölçekli uygulamalar için uygundur.

Node.js ve Express.js, npm (Node Package Manager) aracılığıyla sunulan geniş bir middleware ve modül ekosistemine sahiptir. Bu geniş paket kütüphanesi, kimlik doğrulama, giriş doğrulama ve dosya işlemleri gibi çeşitli işlevleri hızla uygulamayı sağlar, tekerleği yeniden icat etmeye gerek kalmadan.

MongoDB ile Veritabanı Entegrasyonu

MongoDB, JSON benzeri esnek bir formatta (BSON) veri saklayan bir NoSQL veritabanıdır. Bu esneklik, uygulama gereksinimleri değiştikçe veritabanının evrimleşmesini ve ölçeklenmesini kolaylaştırır. MongoDB’nin döküman odaklı yapısı, JavaScript ve JSON ile iyi uyum sağlar, Node.js ve Express ile kesintisiz entegrasyon sağlar.

MongoDB’nin en önemli avantajlarından biri ölçeklenebilirliğidir. Sharding aracılığıyla yatay ölçeklemeyi destekler ve verilerin birden fazla sunucuya dağıtılmasına olanak tanır. Bu özellik, uygulamanın artan yükü ve büyüyen veri hacimlerini performanstan ödün vermeden yönetmesini sağlar.

MongoDB ayrıca güçlü sorgulama ve indeksleme yetenekleri sunar, verilerin verimli bir şekilde alınmasını ve işlenmesini sağlar. Aggregation framework'ü, karmaşık veri dönüşümleri ve analitik işlemleri destekler, bu da onu geniş bir uygulama yelpazesi için uygun hale getirir.

Socket.io ile Gerçek Zamanlı İletişim

Sohbet uygulamaları veya canlı bildirimler gibi gerçek zamanlı iletişim gerektiren uygulamalar için Socket.io, MERN stack'e değerli bir eklentidir. Socket.io, sunucu ve istemciler arasında çift yönlü iletişim sağlar, anında güncellemeler ve etkileşimler gerçekleştirir.

Socket.io'nun MERN stack ile entegrasyonu, Node.js ve Express'i sunucu tarafı uygulamalarında kullanarak oldukça basittir. Bu kombinasyon, gerçek zamanlı işlevlerin sorunsuz bir şekilde uygulamaya dahil edilmesini sağlar, kullanıcı etkileşimini ve deneyimini artırır.

  • Zengin Ekosistem: MERN stack uygulamalarının işlevselliğini genişletmek için geniş bir kütüphane, araç ve üçüncü taraf paketi mevcuttur. Bu ekosistem, yaygın gereksinimler için önceden oluşturulmuş çözümler sunarak geliştirme süresini ve çabasını azaltır.

Maliyet Etkinliği ve Ekip Verimliliği

MERN stack, özellikle küçük ve orta ölçekli uygulamalar için maliyet etkinliği ve ekip verimliliği sağlar. Sisteminizde Node.js ve npm'nin yüklü olduğundan emin olun. Değilse, Node.js resmi web sitesinden indirip yükleyin.

Adım 1: Proje Dizininize Gitmek

Terminalinizi açın ve projenizin server klasörüne gidin. Bunu yapmak için cd (change directory) komutunu kullanın. Örneğin:

cd path/to/your/project/server

Adım 2: Gerekli Bağımlılıkları Kurmak

Sunucu tarafı geliştirme için gerekli olan birkaç npm paketini kurmamız gerekiyor. --save bayrağı, bu bağımlılıkların package.json dosyasına kaydedilmesini sağlar.

npm install --save bcryptjs cors dotenv express joi jsonwebtoken mongoose nodemon socket.io uuid

İşte kurduğumuz paketlerin kısa bir açıklaması:

  • bcryptjs: Şifre hashing için.
  • cors: Cross-Origin Resource Sharing'i etkinleştirmek için.
  • dotenv: .env dosyasından çevre değişkenlerini yüklemek için.
  • express: Minimal ve esnek bir Node.js web uygulama çerçevesi.
  • joi: Şema doğrulaması için.
  • jsonwebtoken: Token tabanlı kimlik doğrulama için.
  • mongoose: MongoDB nesne modelleme için.
  • nodemon: Kod değişikliklerinde sunucuyu otomatik olarak yeniden başlatmak için.
  • socket.io: Gerçek zamanlı çift yönlü olay tabanlı iletişim sağlamak için.
  • uuid: Benzersiz kimlikler oluşturmak için.

Adım 3: Paket Kurulumunu Doğrulamak

Yukarıdaki komutu çalıştırdıktan sonra, paketlerin kurulmasını bekleyin. Kurulum başarılı olduktan sonra, proje dizininizde bir node_modules klasörü görmelisiniz. Bu klasör, tüm kurulu paketleri içerir.

Ayrıca package.json dosyanızı kontrol ederek tüm bağımlılıkların doğru bir şekilde listelendiğinden emin olabilirsiniz. node_modules klasörü eksikse, kurulum komutunu yeniden çalıştırın:

npm install

Adım 4: İlk Sunucu Dosyasını Ayarlamak

server dizininde index.js adında bir dosya oluşturun. Bu dosya, sunucunuzun giriş noktası olarak hizmet edecektir.

Adım 5: Nodemon'u Yapılandırmak

Kod değişikliklerinde sunucunun otomatik olarak yeniden başlatılmasını sağlamak için package.json dosyasına bir script ekleyeceğiz. Aşağıdaki scripti ekleyin:

"scripts": {
  "start": "nodemon index.js"
}

Bu script, sunucuyu şu komutla başlatmanızı sağlar:

npm run start

Nodemon, index.js dosyasındaki değişiklikleri izler ve sunucuyu otomatik olarak yeniden başlatır.

Adım 6: İlk Sunucu Kurulumu

index.js dosyasını açın ve aşağıdaki temel sunucu kurulum kodunu ekleyin:

const express = require('express');
const cors = require('cors');
const dotenv = require('dotenv');
const mongoose = require('mongoose');
const socketio = require('socket.io');
const http = require('http');

dotenv.config();

const app = express();
const server = http.createServer(app);
const io = socketio(server);

// Middleware
app.use(cors());
app.use(express.json());

// Temel route
app.get('/', (req, res) => {
  res.send('Server is running');
});

// Socket.io bağlantısı
io.on('connection', (socket) => {
  console.log('New WebSocket connection');
});

const PORT = process.env.PORT || 5000;

mongoose.connect(process.env.MONGO_URI, {
  useNewUrlParser: true,
  useUnifiedTopology: true,
}).then(() => {
  server.listen(PORT, () => console.log(`Server running on port ${PORT}`));
}).catch((error) => {
  console.error('Database connection error:', error);
});

Basit Bir Express Sunucusu Kurulumu

Bu eğitimde, en son ES module syntax kullanarak basit bir Express sunucusunun nasıl kurulacağını anlatacağız. Bu detaylı kılavuz, olası yanlışlıkları ele alacak ve tüm adımların yazılım geliştirme alanında beklenen titizlik ve standartları yansıtmasını sağlayacaktır.

Gereksinimler

Başlamadan önce, bilgisayarınızda Node.js'in yüklü olduğundan emin olun. Node.js resmi web sitesinden indirebilirsiniz. Ayrıca, JavaScript ve Node.js hakkında temel bilgiye sahip olmalısınız.

Adım 1: Projeyi Başlatma

Öncelikle, projeniz için yeni bir dizin oluşturun ve içine gidin. Ardından, yeni bir Node.js projesi başlatın:

mkdir my-express-server
cd my-express-server
npm init -y

Bu, varsayılan ayarlarla bir package.json dosyası oluşturacaktır.

Adım 2: ES Modules'ı Etkinleştirme

Varsayılan olarak, Node.js CommonJS syntax (require) kullanır. ES module syntax (import) kullanmak için package.json dosyasını değiştirmemiz gerekiyor:

{
  "type": "module",
  "dependencies": {
    // bağımlılıklarınız burada
  }
}

"type": "module" eklemek, require yerine import ifadelerini kullanmamıza olanak tanır.

Adım 3: Bağımlılıkları Yükleme

Express sunucumuz için birkaç paket gerekecek:

  • express: Node.js için bir web framework.
  • cors: Cross-Origin Resource Sharing'i (CORS) ele almak için bir middleware.
  • dotenv: .env dosyasından ortam değişkenlerini yüklemek için bir modül.

Bu paketleri npm kullanarak yükleyin:

npm install express cors dotenv

Adım 4: .env Dosyasını Oluşturma

Projenizin kök dizininde ortam değişkenlerini depolamak için bir .env dosyası oluşturun:

touch .env

.env dosyasına şu içeriği ekleyin:

API_PORT=5002

Adım 5: Sunucuyu Kurma

server.js adında yeni bir dosya oluşturun ve Express sunucunuzun temel yapısını ayarlayın:

import express from 'express';
import http from 'http';
import cors from 'cors';
import dotenv from 'dotenv';

dotenv.config();

const app = express();
const port = process.env.API_PORT || 3000;

app.use(express.json());
app.use(cors());

app.get('/', (req, res) => {
  res.send('Hello, here is your server');
});

const server = http.createServer(app);

server.listen(port, () => {
  console.log(`Server is listening on port ${port}`);
});

Açıklama

  1. Import İfadeleri: Gerekli modülleri (express, http, cors, dotenv) ES module syntax kullanarak import ediyoruz.
  2. Ortam Değişkenleri: Ortam değişkenlerini dotenv kullanarak yapılandırıyoruz.
  3. Middleware:
    • express.json() gelen JSON isteklerini ayrıştırmak için.
    • cors() CORS'u etkinleştirmek için.
  4. Temel Route: Basit bir GET route ayarlıyoruz ve bir mesajla yanıt veriyoruz.
  5. Sunucu Başlatma: HTTP sunucusu oluşturuyoruz ve belirtilen portta dinlemesini sağlıyoruz.

Adım 6: Başlatma Komutunu Ekleme

Sunucunun değişiklikleri algılayarak otomatik olarak yeniden başlatılmasını sağlayan nodemon kullanarak package.json dosyasını güncelleyin:

{
  "scripts": {
    "start": "nodemon server.js"
  },
  "dependencies": {
    "express": "^4.17.1",
    "cors": "^2.8.5",
    "dotenv": "^10.0.0"
  },
  "devDependencies": {
    "nodemon": "^2.0.7"
  },
  "type": "module"
}

nodemon'ı bir geliştirme bağımlılığı olarak yükleyin:

npm install --save-dev nodemon

Adım 7: Sunucuyu Çalıştırma

Sunucuyu şu komutla başlatın:

npm start

Terminalinizde Server is listening on port 5002 mesajını görmelisiniz. Tarayıcınızı açın ve http://localhost:5002 adresine giderek Hello, here is your server yanıtını görün.

Postman İndirme ve Kurulum

Başlamak için, Postman'i resmi web sitesinden indirin ve kurun:

  1. Postman'in resmi web sitesi adresine gidin.
  2. İşletim sisteminiz için uygun sürümü indirin.
  3. Platformunuz için kurulum talimatlarını izleyin.

Kurulum tamamlandıktan sonra Postman'i açın. Arayüz, istekler oluşturmak ve yönetmek için çeşitli araçlar ve seçenekler sunacaktır.

Postman Kurulumu

API'nizi test etmeye başlamadan önce Postman'in düzenini ve özelliklerini anlamak önemlidir:

  1. Workspaces: Postman, istekleri workspaceler içinde organize eder. Yeni bir workspace oluşturun veya mevcut birini kullanın. Workspace, ilgili istekleri ve koleksiyonları verimli bir şekilde yönetmeye yardımcı olur.
  2. Collections: Koleksiyonlar, ilgili istek gruplarıdır. İsteklerinizi daha iyi yönetim için koleksiyonlar halinde düzenleyin.

API İstekleri Oluşturma ve Test Etme

Yeni Bir İstek Ekleme

Postman'de yeni bir istek eklemek için:

  1. Yeni bir istek sekmesi açmak için + düğmesine tıklayın.
  2. Test etmek istediğiniz HTTP yöntemini (GET, POST, PUT, DELETE vb.) seçin.

İstek URL'sini Tanımlama

Bu örnek için, yerel olarak 5002 portunda çalışan bir sunucumuz olduğunu ve login API endpoint'ini test etmek istediğimizi varsayalım.

  1. URL'yi http://localhost:5002/api/login olarak ayarlayın.
  2. HTTP yöntemini (örneğin, POST) seçin.

İsteği Yapılandırma

Postman, isteğin çeşitli yönlerini yapılandırmanıza olanak tanır:

  1. Headers: Gerekli header'ları ekleyin, örneğin Content-Type olarak application/json ayarlayın.
  2. Body: POST istekleri için isteğin yükünü body kısmına ekleyin. raw seçeneğini seçin ve formatı JSON olarak ayarlayın.

Bir login isteği için örnek payload:

{
  "username": "exampleUser",
  "password": "examplePassword"
}

İsteği Gönderme

İsteği yapılandırdıktan sonra:

  1. Send düğmesine tıklayın.
  2. Sunucu tarafından döndürülen yanıtı inceleyin.

Yanıtları Yönetme

Postman, yanıt verilerini, durum kodlarını, header'ları ve yanıt gövdesini görüntüler. Bu bilgi, API'nizin beklendiği gibi çalıştığını doğrulamak için çok önemlidir.

Adım Adım Rehber

1. Controllers Klasörünü Oluşturun

Öncelikle, src dizini içinde bir controllers klasörü oluşturun. Bu klasör, tüm kontrolör mantığımızı barındıracaktır.

2. Auth Kontrolörlerini Oluşturun

controllers klasörü içinde bir auth klasörü oluşturun. Bu auth klasörü içinde iki dosya oluşturun: postLogin.js ve postRegister.js.

postLogin.js:

// src/controllers/auth/postLogin.js

export const postLogin = async (req, res) => {
    try {
        // Authentication logic here
        res.status(200).json({ message: "This is login route" });
    } catch (error) {
        res.status(500).json({ error: "Internal Server Error" });
    }
};

postRegister.js:

// src/controllers/auth/postRegister.js

export const postRegister = async (req, res) => {
    try {
        // Registration logic here
        res.status(200).json({ message: "This is register route" });
    } catch (error) {
        res.status(500).json({ error: "Internal Server Error" });
    }
};

3. Kontrolörleri Birleştirin

Tüm kontrolörleri birleştirmek için controllers klasöründe bir controllers.js dosyası oluşturun.

controllers.js:

// src/controllers/controllers.js

import { postLogin } from './auth/postLogin';
import { postRegister } from './auth/postRegister';

export { postLogin, postRegister };

4. Yolları Tanımlayın

src klasöründe bir routes klasörü oluşturun. Bu klasör içinde kimlik doğrulama yolları için bir auth.js dosyası oluşturun.

auth.js:

// src/routes/auth.js

import express from 'express';
import { postLogin, postRegister } from '../controllers/controllers';

const router = express.Router();

router.post('/login', postLogin);
router.post('/register', postRegister);

export default router;

5. Yolları Uygulamaya Entegre Edin

Ana uygulama dosyanızı (örneğin, app.js) tanımlı yolları kullanacak şekilde değiştirin.

app.js:

// app.js

import express from 'express';
import authRoutes from './src/routes/auth';

const app = express();

app.use(express.json());
app.use('/api/auth', authRoutes);

const PORT = process.env.PORT || 5000;
app.listen(PORT, () => {
    console.log(`Server is running on port ${PORT}`);
});

  • Login Route:

    • URL: http://localhost:5000/api/auth/login
    • Method: POST
    • Response: { "message": "This is login route" }

  • Register Route:

    • URL: http://localhost:5000/api/auth/register
    • Method: POST
    • Response: { "message": "This is register route" }

1. Veritabanı Yapılandırması

MongoDB'ye bağlanmak için config/db.js dosyasını oluşturun:

const mongoose = require('mongoose');

const connectDB = async () => {
    try {
        await mongoose.connect(process.env.MONGO_URI, {
            useNewUrlParser: true,
            useUnifiedTopology: true,
            useCreateIndex: true,
        });
        console.log('MongoDB Connected');
    } catch (err) {
        console.error(err.message);
        process.exit(1);
    }
};

module.exports = connectDB;

3. Auth Controller

Kayıt ve giriş işlemlerini yönetmek için controllers/authController.js dosyasını oluşturun:

const User = require('../models/User');
const bcrypt = require('bcryptjs');
const jwt = require('jsonwebtoken');

exports.register = async (req, res) => {
    const { username, email, password } = req.body;

    try {
        let user = await User.findOne({ email });
        if (user) {
            return res.status(400).json({ msg: 'User already exists' });
        }

        user = new User({
            username,
            email,
            password,
        });

        const salt = await bcrypt.genSalt(10);
        user.password = await bcrypt.hash(password, salt);

        await user.save();

        const payload = {
            user: {
                id: user.id,
            },
        };

        jwt.sign(
            payload,
            process.env.JWT_SECRET,
            { expiresIn: 3600 },
            (err, token) => {
                if (err) throw err;
                res.json({ token });
            }
        );
    } catch (err) {
        console.error(err.message);
        res.status(500).send('Server error');
    }
};

exports.login = async (req, res) => {
    const { email, password } = req.body;

    try {
        let user = await User.findOne({ email });
        if (!user) {
            return res.status(400).json({ msg: 'Invalid credentials' });
        }

        const isMatch = await bcrypt.compare(password, user.password);
        if (!isMatch) {
            return res.status(400).json({ msg: 'Invalid credentials' });
        }

        const payload = {
            user: {
                id: user.id,
            },
        };

        jwt.sign(
            payload,
            process.env.JWT_SECRET,
            { expiresIn: 3600 },
            (err, token) => {
                if (err) throw err;
                res.json({ token });
            }
        );
    } catch (err) {
        console.error(err.message);
        res.status(500).send('Server error');
    }
};

4. Doğrulama Middleware

İstek verilerini doğrulamak için middlewares/validateRequest.js dosyasını oluşturun:

const { body, validationResult } = require('express-validator');

exports.validateRegister = [
    body('username').notEmpty().withMessage('Username is required'),
    body('email').isEmail().withMessage('Please include a valid email'),
    body('password').isLength({ min: 6 }).withMessage('Password must be at least 6 characters'),
    (req, res, next) => {
        const errors = validationResult(req);
        if (!errors.isEmpty()) {
            return res.status(400).json({ errors: errors.array() });
        }
        next();
    },
];

exports.validateLogin = [
    body('email').isEmail().withMessage('Please include a valid email'),
    body('password').exists().withMessage('Password is required'),
    (req, res, next) => {
        const errors = validationResult(req);
        if (!errors.isEmpty()) {
            return res.status(400).json({ errors: errors.array() });
        }
        next();
    },
];

5. Auth Routes

Kimlik doğrulama rotalarını tanımlamak için routes/authRoutes.js dosyasını oluşturun:

const express = require('express');
const { register, login } = require('../controllers/authController');
const { validateRegister, validateLogin } = require('../middlewares/validateRequest');
const router = express.Router();

router.post('/register', validateRegister, register);
router.post('/login', validateLogin, login);

module.exports = router;

6. Sunucu Kurulumu

Express sunucusunu kurmak için server.js dosyasını güncelleyin:

const express = require('express');
const connectDB = require('./config/db');
const authRoutes = require('./routes/authRoutes');
require('dotenv').config();

const app = express();

// Connect Database
connectDB();

// Init Middleware
app.use(express.json());

app.get('/', (req, res) => res.send('API Running'));

// Define Routes
app.use('/api/auth', authRoutes);

const PORT = process.env.PORT || 5000;

app.listen(PORT, () => console.log(`Server started on port ${PORT}`));

Joi ile Validation

Giriş

Veri doğrulama, güvenli ve güvenilir uygulamalar geliştirirken önemli bir adımdır. Müşterilerden alınan verilerin beklendiği gibi olmasını sağlar ve potansiyel hataları ve güvenlik açıklarını önler. Bu eğitimde, bir Express uygulamasında istek verilerini doğrulamak için güçlü bir şema tanımlama dili ve veri doğrulayıcısı olan Joi'yi nasıl kullanacağımızı inceleyeceğiz.

Joi Nedir?

Joi, JavaScript nesnelerini doğrulamak için kapsamlı bir kütüphanedir. Geliştiricilerin verileri için şemalar tanımlamalarına, gerekli yapıyı, türleri ve kısıtlamaları belirtmelerine olanak tanır. Joi, gelen verilere kurallar uygulamayı kolaylaştırır ve doğrulama başarısız olduğunda anlamlı hata mesajları sağlar.

Ortamı Ayarlamak

Başlamadan önce Node.js ve npm'nin kurulu olduğundan emin olun. Yeni bir Express projesi oluşturabilir veya mevcut birini kullanabilirsiniz. Gerekli paketleri yükleyin:

npm install express joi express-joi-validation

Doğrulama Şemalarını Oluşturmak

Joi kullanarak kullanıcı kaydı ve giriş için şemalar oluşturacağız. Bu şemalar, istek verilerinde beklediğimiz alanlar için kuralları tanımlayacaktır.

Register Şeması

const Joi = require('joi');

const registerSchema = Joi.object({
  username: Joi.string().alphanum().min(3).max(30).required(),
  password: Joi.string().min(6).required(),
  email: Joi.string().email().required()
});

module.exports = { registerSchema };

Login Şeması

const Joi = require('joi');

const loginSchema = Joi.object({
  email: Joi.string().email().required(),
  password: Joi.string().min(6).required()
});

module.exports = { loginSchema };

Joi'yi Express ile Entegre Etmek

Sonraki adımda, Joi'yi gelen istekleri doğrulamak için Express uygulamamızla entegre edeceğiz. Doğrulama ara yazılımını yönetmek için express-joi-validation paketini kullanacağız.

Express ve Doğrulayıcıları Ayarlamak

const express = require('express');
const { createValidator } = require('express-joi-validation');
const { registerSchema, loginSchema } = require('./schemas');

const app = express();
const validator = createValidator();

app.use(express.json());

Doğrulama ile Rotaları Tanımlamak

Kullanıcı kaydı ve giriş için rotaları tanımlıyoruz ve Joi doğrulama ara yazılımını uyguluyoruz.

const registerController = (req, res) => {
  // Kullanıcıyı kaydetmek için özel mantık
  res.send('User registered successfully');
};

const loginController = (req, res) => {
  // Kullanıcıyı giriş yapmak için özel mantık
  res.send('User logged in successfully');
};

app.post('/register', validator.body(registerSchema), registerController);
app.post('/login', validator.body(loginSchema), loginController);
